Telegraph System & Morse Code

The telegraph was the first electrical long-distance communication system — the ancestor of all modern data networks.

The telegraph transmitted electrical signals over long distances, separating messages into coded symbols (dots and dashes) through a physical wire medium — the foundation of modern data communications.

  • Sender: Encodes a message into electrical pulses using Morse code via the telegraph key.
  • Channel: Carries the pulses through a wire across vast distances.
  • Receiver: Converts the pulses back into readable text at the destination station.
  • Protocol: Morse code — an agreed set of rules for timing and interpretation of pulses.
  • Legacy: Directly inspired modern packet-switched networks and digital encoding schemes.

📟 Telegraph & Morse Code
The telegraph was an early long-distance communication device that used electrical signals or pulses through wires.
The telegraph key was used to make and break the electrical circuit, generating short and long pulses (dots and dashes).
The receiver at the other end interpreted the clicks and signals into recognizable messages.
Morse code is a system of dots and dashes used to represent letters, numbers, and punctuation.
Students are required to memorise the first 15 letters of the Morse code alphabet.
🛰️ Satellite Communication Systems
Basic components include: ground station, uplink, satellite transponder, and downlink.
Signal transmission travels from the sender (ground station) uplink to the satellite, then downlink to the receiver.
The satellite transponder handles frequency conversion and signal amplification.
Three main orbit types: GEO (Geostationary Earth Orbit), MEO (Medium Earth Orbit), and LEO (Low Earth Orbit).
Advantages include wide coverage area and broadcasting capability to large regions simultaneously.
Limitations include signal latency (especially GEO), atmospheric attenuation, and high deployment cost.
📶 Hub-Based Ethernet Extended to Wireless (APs & PoE)
Hub-based Ethernet operates as a shared medium — all connected devices share the same bandwidth.
CSMA/CD (Carrier Sense Multiple Access / Collision Detection) is used in hub-based Ethernet to manage collisions.
CSMA/CA (Collision Avoidance) is used in wireless networks to reduce the chance of packet collision.
An Access Point (AP) connects the wired Ethernet segment to the wireless network segment.
Power over Ethernet (PoE) delivers electrical power through the network cable, removing the need for separate power supplies for APs.
Deployment strategies include AP placement for coverage, structured cabling, and network segmentation.
Critical analysis: performance is impacted by interference, AP density, and channel overlap in wireless extensions.
📜 Evolution of Data Communication
Early communication systems included smoke signals, semaphore flags, and light-based (optical) systems.
These systems encoded information through visual patterns, positions, or light pulses visible at a distance.
Transition to electrical communication introduced the telegraph, telephone, and early radio systems.
Modern digital networks evolved from analogue electrical systems through digitisation, packet switching, and the internet.
Core principles remain constant: encoding, signalling, transmission, a sender, a receiver, and a medium.
Early systems directly influenced protocol design, error handling, and layered architectures used in modern networks.
🔐 Pretty Good Privacy (PGP) & Secure Data Communication
PGP (Pretty Good Privacy) is a cryptographic system used to secure emails and files through encryption and digital signatures.
Encryption converts plaintext into ciphertext; decryption reverses the process using the correct key.
Digital signatures verify the identity of the sender and ensure the message has not been tampered with.
Key management uses a public key (shared freely) and a private key (kept secret by the owner).
PGP combines symmetric cryptography (for speed) and asymmetric cryptography (for secure key exchange).
Security goals: Confidentiality (only the intended recipient reads it), Integrity (data unchanged), Authentication (verified sender), Non-repudiation (sender cannot deny sending).
Real-world applications include secure email, file encryption, and software signing; limitations include complexity of key management and revocation.
